Arquitetura Técnica

Visão detalhada da arquitetura em camadas, stack tecnológico e estratégias de performance do SIGA

Arquitetura em Camadas

4 Camadas Integradas

O SIGA utiliza uma arquitetura em camadas bem definida, garantindo separação de responsabilidades e manutenção simplificada.

1

Camada de Apresentação

  • Interface Web responsiva
  • API REST com documentação OpenAPI
  • Componentes UI reutilizáveis
2

Camada de Negócio

  • Módulo de Materiais e Categorias
  • Módulo de Almoxarifados e Localizações
  • Módulo de Operações e Movimentações
  • Módulo de Segurança e Auditoria
  • Módulo de Notificações
3

Camada de Dados

  • PostgreSQL com extensões
  • Triggers automáticos e funções PL/pgSQL
  • Views materializadas para relatórios
  • Particionamento temporal (mensal/trimestral/semestral)
  • JSONB para flexibilidade de esquema
4

Camada de Infraestrutura

  • Backup automatizado diário
  • Monitoramento e alertas
  • Replicação master-slave
  • Segurança em camadas (HTTPS, firewall, WAF)
Stack Tecnológico

Tecnologias em Detalhe

Cada tecnologia foi selecionada para atender requisitos de performance, segurança e escalabilidade do SIGA.

PostgreSQL 15+

Banco relacional robusto com extensões avançadas. Particionamento nativo, JSONB, triggers, funções PL/pgSQL e LISTEN/NOTIFY.

REST API

Arquitetura de microsserviços com endpoints RESTful, documentação OpenAPI/Swagger e versionamento de API.

JWT Authentication

Autenticação stateless com access tokens (15min) e refresh tokens (7 dias). Rotação automática de tokens.

RBAC Hierárquico

Controle de acesso baseado em papéis com 5 perfis, herança de permissões e controle granular por módulo.

LISTEN/NOTIFY

Mecanismo nativo do PostgreSQL para notificações em tempo real sem polling, integrado com WebSocket no frontend.

Particionamento

Estratégia temporal para tabelas de alto volume: operações (trimestral), auditoria (mensal), notificações (mensal).

Banco de Dados

Arquitetura do Banco de Dados

PostgreSQL 15+ como fundação, com estratégias avançadas de particionamento, indexação e automação.

Normalização Controlada

3a forma normal com desnormalização estratégica para consultas frequentes, equilibrando integridade e performance.

Extensibilidade

Campos JSONB para atributos dinâmicos sem alteração estrutural, permitindo evolução do esquema sem migração.

Rastreabilidade

Auditoria completa com dados antes/depois em todas as operações, garantindo conformidade e transparência.

Segurança

Segurança da Arquitetura

Múltiplas camadas de proteção garantem a integridade e confidencialidade dos dados do SIGA.

Autenticação

JWT com access token (15min) + refresh token (7 dias). Rotação automática de tokens para segurança contínua.

Autorização

RBAC hierárquico com 5 perfis. Herança de permissões com controle granular por módulo e operação.

Criptografia

HTTPS/SSL/TLS em trânsito. Bcrypt para senhas. Campos sensíveis criptografados em repouso.

Auditoria

Log completo de todas as operações com dados antes/depois, timestamp, usuário responsável e endereço IP.

Proteção

Validação de entrada (SQL injection, XSS), rate limiting, CORS configurado e headers de segurança.

Performance

Otimizações de Performance

Estratégias avançadas para garantir tempos de resposta rápidos e uso eficiente de recursos.

Views Materializadas

Para relatórios complexos com refresh automático por triggers, evitando recálculo a cada consulta.

Cache Inteligente

Cache de consultas frequentes com invalidação por trigger, garantindo dados sempre atualizados.

Connection Pooling

PgBouncer para gerenciamento eficiente de conexões, reduzindo overhead e melhorando concorrência.

Query Optimization

Análise com EXPLAIN ANALYZE para otimização contínua de consultas e planos de execução.

Lazy Loading

Carregamento sob demanda de dados em listas paginadas, otimizando uso de rede e memória.
