JWT

Autenticação e Sessão

Mecanismos robustos de autenticação baseados em tokens JWT com proteção em múltiplas camadas

Access Token

JWT com expiração de 15 minutos. Contém: user_id, perfil, unidade, permissoes.

Refresh Token

Validade de 7 dias. Armazenado como HttpOnly cookie. Rotação automática a cada uso.

Sessão

Registro de IP, user-agent e timestamp. Timeout por inatividade (30 min). Máximo 3 sessões simultâneas.

Proteção

Rate limiting em login (5 tentativas/5min). Bloqueio temporário após falhas. Notificação de login em novo dispositivo.

Fluxo de Autenticação

1

Login Request

Usuário envia credenciais (email + senha)

2

Validação

Bcrypt hash check + rate limiting + verificação de bloqueio

3

Geração de Tokens

Access Token (JWT 15min) + Refresh Token (7 dias, HttpOnly cookie)

4

Registro de Sessão

IP, user-agent, timestamp registrados na tabela de sessões

5

Requisições Autenticadas

Access Token no header Authorization: Bearer <token>

6

Renovação Automática

Refresh Token rotaciona automaticamente gerando novo par de tokens

Sistema de Auditoria

Toda ação no sistema é registrada automaticamente via triggers do banco de dados

Quem

user_id, nome, perfil, unidade

O que

Tabela, operação (INSERT / UPDATE / DELETE), campo alterado

Quando

Timestamp com precisão de microssegundos

Onde

Endereço IP, user-agent

Contexto

Dados anteriores (JSONB), dados posteriores (JSONB)

Armazenamento

Tabela log_auditoria particionada mensalmente para performance ótima em consultas e manutenção eficiente do armazenamento.

Retenção

5 anos no banco ativo. Exportação automática para arquivo após 5 anos. Descarte após 10 anos.

Imutabilidade

Registros de auditoria são somente INSERT - não podem ser alterados ou excluídos. Garantia de integridade total dos registros.

Triggers Automáticos

Toda ação no sistema é capturada automaticamente via triggers do PostgreSQL. Não depende da aplicação - a auditoria acontece no nível do banco de dados.

Defesa em Profundidade

Proteção de Dados

Segurança em múltiplas camadas para proteção completa dos dados

Em Trânsito

  • HTTPS/SSL/TLS obrigatório em todas as conexões
  • Certificados validados e renovados automaticamente
  • HSTS habilitado para prevenir downgrade de protocolo

Em Repouso

  • Senhas com Bcrypt (salt único por usuário)
  • Campos sensíveis criptografados no banco de dados
  • Backup criptografado com chave de segurança

Na Aplicação

  • Validação de entrada (SQL injection, XSS)
  • CORS configurado com origens permitidas
  • CSP headers e rate limiting ativo
